# Plan 6.2: Streamlit UI Frontend

## Objective
Develop a Streamlit web application that serves as the user interface for the Neuro-Symbolic Trading System. Users can input ticker symbols and date ranges, and view the trading signals generated by the Flask API.

## Context
- .gsd/SPEC.md
- src/app.py (Flask API from wave 1)

## Tasks

<task type="auto">
  <name>Develop Streamlit Dashboard</name>
  <files>
    - ui/dashboard.py
    - requirements.txt
  </files>
  <action>
    - Add `streamlit` and `requests` to `requirements.txt` if not present.
    - Create the `ui` directory if it doesn't exist.
    - Build `ui/dashboard.py` containing the Streamlit application.
    - Create input fields for Ticker Symbol (e.g., TSLA), Start Date, and End Date (or a predefined period).
    - On button click, send a POST request to the Flask backend (`http://localhost:5000/predict`).
    - Parse the JSON response and visualize the trading signals (e.g., using a Streamlit dataframe or line chart highlighting Buy/Sell zones).
  </action>
  <verify>streamlit run ui/dashboard.py</verify>
  <done>Streamlit dashboard runs successfully, communicates with the local Flask API, and displays prediction results visually.</done>
</task>

## Success Criteria
- [ ] User can input a ticker and fetch predictions.
- [ ] The dashboard successfully graphs or tablets the valid structural signals returned from the backend.
